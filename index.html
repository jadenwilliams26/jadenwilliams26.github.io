Combined Narrative

Briefly describe the artifact. The artifact is a Python-based thermostat system designed for a reptile enclosure. It was originally developed in early 2025 as a functional temperature and humidity controller using a Raspberry Pi, DHT22 sensor, heater, and misting system. Over the course of this project, the system was significantly enhanced in three stages: software design and engineering improvements (Category One), advanced algorithm and data structure integration (Category Two), and MongoDB database integration for dynamic setpoint retrieval (Category Three). Justify the inclusion of the artifact in your ePortfolio. This project was selected for my ePortfolio because it demonstrates my ability to design, improve, and extend a real-world IoT solution through multiple layers of development. It showcases my skills in structured software design, hardware integration, GPIO-based input handling, algorithmic problem-solving, and database-driven customization. Across the three enhancement stages, I applied best practices in class-based architecture, efficient data handling, and database interaction — all while managing hardware components. The artifact’s progression clearly reflects growth in both coding quality and system functionality, making it an excellent representation of my software development capabilities. How was the artifact improved? Category One: Software Design and Engineering — Refactored the code into a modular, class-based architecture with Thermostat, Controller, and DisplayManager classes. Added structured error handling, improved commenting, and clearer separation of responsibilities for maintainability and scalability. Category Two: Algorithms and Data Structures — Introduced GPIO-based menu navigation using physical buttons for adjusting temperature and humidity setpoints. Implemented algorithmic logic for LED-based environmental alerts, improving real-time responsiveness to sensor readings. Category Three: Databases — Integrated a local MongoDB database to store species-specific temperature and humidity requirements. The system now automatically pulls correct setpoints when a species is selected, eliminating the need for manual reconfiguration and making the system more adaptable to various reptile species. Did you meet the course outcomes you planned to meet with this enhancement in Module One? Yes. The enhancements addressed planned course outcomes, including designing and evaluating computing solutions, demonstrating innovative techniques and tools, and creating software that meets specific industry needs. The database integration also aligned with outcomes related to managing data effectively and ensuring solutions are adaptable to future requirements. Reflect on the process of enhancing and modifying the artifact. Enhancing this artifact taught me the importance of planning for modularity early in development. While adding new features, I learned how clear class responsibilities and well-documented code help streamline the integration of new functionalities — especially when hardware components are involved. The GPIO-based input system required careful algorithmic planning to prevent misreads and ensure user-friendliness. Integrating MongoDB added another layer of complexity, as I had to design data structures that were intuitive, scalable, and reliable for retrieving species-specific settings. The biggest challenges were ensuring smooth hardware/software interaction, particularly when switching between modes and updating setpoints in real time, and maintaining clean communication between the database and the thermostat logic. Overcoming these obstacles strengthened my ability to design full-stack IoT systems and deepened my understanding of how to merge software engineering principles with physical device control.
