The initial enhancements focused on creating a maintainable and scalable structure, implementing clear separation of concerns, and preparing the code for future integration of additional features.

I selected this artifact because it demonstrates my ability to structure complex software systems for both clarity and expansion. These enhancements met my planned outcomes for Module One by showing my ability to apply object-oriented design principles to a hardware-software integrated system. I learned the value of designing with future integration in mind, which reduced the complexity of later adding algorithmic and database features. Category Two: Algorithms and Data Structures

In the second enhancement phase, I expanded the artifact by implementing real-time setpoint adjustment using physical buttons connected via GPIO. The BUTTON_MODE, BUTTON_UP, and BUTTON_DOWN pins allow the user to cycle between temperature and humidity adjustment modes and increment or decrement the setpoints. I also incorporated RED_LED and BLUE_LED indicators to signal alert conditions, such as when readings deviate from desired ranges.

This enhancement was chosen because it illustrates my ability to apply algorithms and data structures to hardware-interfaced applications. I implemented efficient control loops that handle input debouncing, state tracking, and conditional logic for alerts. The system’s ability to update setpoints in-memory without interrupting the control process demonstrates strong event-driven programming skills.

I met my Module One course outcomes by showcasing my capability to design algorithms that interact with hardware in real-time while maintaining reliability. I learned to manage state efficiently and ensure responsiveness, even under the constraints of GPIO input timing. The primary challenge was ensuring smooth transitions between modes and avoiding false inputs due to button bounce, which I resolved through software debouncing techniques.

Category Three: Databases

In the final enhancement, I integrated a local MongoDB database to store and retrieve species-specific temperature and humidity setpoints. When a species is selected in the system, the thermostat automatically queries the database and applies the retrieved setpoints without requiring manual configuration.

This enhancement highlights my database integration skills, including connecting to MongoDB, executing queries, and incorporating retrieved data into the system’s operational logic. It showcases my ability to combine persistent data storage with hardware control, allowing the system to adapt dynamically to different operational requirements.

The enhancement fulfilled my planned course outcomes by demonstrating proficiency in database management and its application within a larger software-hardware system. I learned how to optimize database queries to minimize latency and ensure seamless integration with real-time control processes. The main challenge was ensuring that the database access did not interrupt the timing-sensitive GPIO control loop, which I solved by loading setpoints before initiating the main control cycle.
